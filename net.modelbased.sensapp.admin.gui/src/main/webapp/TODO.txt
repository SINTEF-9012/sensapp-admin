Visualisation/Sensor

  - Bouton EXPORT (senml/json, senml/xml, CSV, ...)
  - utiliser un tag "default_widget" pour spécifier quel widget graphique utiliser pour un capteur

    - (un bouton "configure" pour slectioner ces paramètres)
    - bouton refresh sur les visualisation

- Visualisation/Composite

  - idem visualisation
  - création d'un widget de visu pour chaque capteur contenu
  - export: (comme les atomique ++ KML, GPX)
  - utiliser un tag "lat_sensor" et un tag "lon_sensor" pour identifier des données a afficher sous forme de carte. Si ces 2 infos sont présente, les 2 capteurs ne sont pas affcihés en tant que tel mais plutôt graphé sur une carte Goolge Maps.

- Dashboard

  - sélectionner les capteurs, les widgets a utiliser
  - exporte une code javascript stand-alone qu'on peut copier/coller dans une page web externe 


Widget de visualisation:

  - atomic: 
    - need: google map, intégrale comme une "Area chart", mécanisme de "polling" pour rafraichir les graphes 
    - opt:

      - logarithmique échelle
